Front-End Framework and images

In week 7, we took a look at what is a Front-End Framework and each chose one to discuss. We then discussed our skills with images and how to resize/crop them

Front-End Frameworks

I'll be doing Milligram. Milligram is a minimalist CSS framework that is currently on v1.3.0, which was released on 01/26/2017. This isn't just a UI framework, but is optimized to be quick and clean. When Gzipped, Milligram is only 2 kb, and around 10kb without compression. It uses font-family Roboto. The main issue with Milligram is that it isn't a particularly rich framework with a lot of content to work with. The more complex styled components are a bit lacking, which generally means that you have to write more of your own css or other sass files to get where you want your website to be. This kind of defeats the purpose of having a front-end framework, in my opinion, but it really depends on the user. Overall, it is a slick, quick, and clean framework that is good building a simple site with a quick loading time.

Images!

I've been working with images for a long time, editing them for work and my own personal use. I've used Photoshop whenever I have access to it, but it tends to be a bit expensive. Due to this, I tend to use GNU Image Manipulation Program (GIMP) for more advanced things and PhotoDemon, a portable, lightweight, and open source program for simple image editing. I usually use JPEG files as they tend to be smaller, but when you are editing them over and over it tends to degrade the image over time. I tend to use PNG files a bit more. Like a GIF, they allow for irregular image sizes by the use of transparaent space. JPGs tend to force a rectangle with visual whitepsace, while a PNG file will let you have transparent space to place images on top of one another or on a background. This makes them good for logos.
