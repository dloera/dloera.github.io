This week we watched a video about Payment Gateways and Web Dev Build Systems.

Payment Gateway

I'm a little late, was busy with work last week unfortunately, but I'm here now. I was interested in WHY payment gateways might or might not be required. So I chose this website as a reference:
Payment gateways are said to be needed to act as a conduit between the buyer and seller through a third-party website/app such as PayPal, Woocommerce, etc. Security of the payment between merchant and customer is important, so the use of a trusted gateway might be considered trustworthy. While they can be expensive to use, nowadays they now offer even more services, such as calculating taxes for you or accepting different forms of payment, or even help with fraud-prevention. In a global economy, this can be very important. They also provide a bit of insulation from having to deal with the security aspects of making a transaction, and can focus on the products for an eCommerce page.

Web Dev Build Systems

I chose to do Webpack, which is an open-sourse JS module bundler to consolidate your JS files. It builds a dependency graph that maps modules that a project needs into one (or more as needed) bundles. The way I understand it is that you tell Webpack which JS files to use, then Webpack creates a hierarchy of files/dependencies needed to create a singular bundle. Without any plugins, Webpack only works with JS and JSON files. With further plugins, you can expands the functionality, such as being able to use Webpack on HTML or CSS Files. There are quite a few options for Webpack, one being mode, which helps to optimize the website either for development or production. The purpose of Webpack, unlike some of the other module bundlers, is to support any module format to handle both resource management and asset management alike. While putting all JS files into one bit file can work, taking into account dependencies, it may not be ideal for how a website is actually loaded. Webpack helps with this.
