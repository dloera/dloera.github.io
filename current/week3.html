Git, Command Line, and Microsoft

In week 3, we covered how to install git and use the command line. Then we discussed how Microsoft purchased Github

Install git and determine your command line interface

I will be using git bash primarily through the VScode editor (alt+`). I have quite a lot of experience using command line and bash terminals. I first started on Windows 3.1 and Unix back when I was a little kid; my mother worked with computers and always had them around, so I had to learn it over the years. Eventually, I found that there are some pretty powerful things you can do with command line through the use of batch files in automation. At work I still use them quite regularly when dealing with a lot of files at once (such as mass rename, mass delete). I have been using for loops in *.bat files for the last 15 years in processing video files, audio files, images, and any other files where I don't want to have to go through things one at a time. It has helped in learning the basics of how a computer runs in the background and I've never had any problems with it.

Microsoft Purchases Github

Link to my article
One of the interesting parts of this article is the concept of ownership for code written through GitHub, and that some think that this will lead to Microsoft co-opting code written through the platform. However, this doesn't quite appear to be an actual concern, as a user does own all of their code in the repository, and can license it how they like (typically under an open-source license such as GPL3. Also, since Microsoft invested $7.5 billion into the platform, the ideal of how they are going to monetize the system further was an interesting part of the article. This was not a cheap acquisition of a company, but Microsoft appears to try to integrate GitHub with Azure and VScode and to what extent will this affect the platform.
