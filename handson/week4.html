Git Workflow and CSS Pages

For Week 4 in Hands-on, we discussed how to add/commit/push our Github repos from the repos that we work with on our personal computers to our repos on the github servers, and then added CSS to our webpages, though it looks much different now.

Git Workflow:

I haven't had any problem with updating my repos with git through the terminal. I have a bit of experience with using the terminal over the years, so doing it isn't anything new, just learning the commands is new, which seem pretty straight forward. Add the file(s), commit it, then push the file to GitHub, though I did run into a brief issue sending the image above to my public folder due to not updating it. Did a quick git pull to update my local files, then a push to send it over so that I can embed the image in this reply. I enjoy having the terminal directly in VS code, but have no problem using the command prompt in windows as well. A git command is able to be used directly in the command prompt, and the only difference from the Git Bash is that it isn't colored.
