Scoping eCommerce Projects

The Lynda.com lecture below discussed the processes of scoping out a project.

ECommerce:

Here is the Lynda.com lecture regarding scoping eCommerce projects

My take on it:

scoping projects:

This is the process of Planning a project before you build it. You must get a clear understanding from the client as to what is desired in a product. Costs can be greater when you must fix a product, instjust of building it correctly the first time. Scaling is important, but being clear in what is desired is important as well. Words have different meanings for different people, so if needed, clarify what a client is seeking rather than assuming. Clarification clears the air from the beginning, so that everytone is on the same page. When planning a project, consider 3 things: Will it be fast, good, and/or cheap. You must choose 2 of the 3 for the client to decide what is important. Cannot have all 3

Different Project Strategies

Waterfall is where there is a list of tasks, in order, that must be completed one by one. It is a linear process, and the biggest problem is if you overlook one problem, it can cause issues down the line. Multi-phase is breaking down the project in to two or more waterfalls, by design. This can help break down a product into subproducts to be developed at the same time by separate developers. Minimum Viable Product (MVP) is the baseline for what a client will accept. The Site can start out simply (for the sake of cost), and the complexity can be built up in time.

Discovery

There are different things to consider in this: What type of product will the merchant be selling? Physical, Digital, or a Service. Is the product ready to go, or does it take time to build. A physical product takes time to ship, which also needs to be discussed. If it is a physical product, then details about the product must be known and provided in a CSV file including dimensions/color/images/etc. If it is a software, what type of licensing requirements are there?

Quantity of the product can also be important. If there is one or a few products to be sold, customizing the pages can be useful and save time. While if there are 100+ products, then the site would need to be built on a template with a CSV file provided by the client, and the client will need to know what is expected of them.

Access to the clients servers is a requirement. If they are not providing you with the tools that you need, then why are you working for them? You will need Admin access to update the site, access to their domain registrars, their hosts, google analytics. Also consider vendor access.

Red flags when discussing details, consider if they have out-of-date software, which may be an issue as software may no longer be support and cause issues. If software needs to be updated, discuss with the client. Charge them for upgrading the software as needed. If they are using a low-quality host, then this can cause issues with how the website will be seen/loaded.

Scaling issues can be an issue that needs to be considered. Client will care about the number of products, traffic on a good host will update your needs as they grow. A bad host will not take your needs into consideration and can cause issues. Sessions hve to remember who the user is, and checkout can be an issue if the business grows quickly.

Payment needs for the client can be an issue. New merchants can be overwhelming, and usin paypal/amazon/apple pay, etc. lets customers use their existing accounts, but can often cost more. Credit cards a necessary, but it really depends on who will be using the site.

Shipping needs can be an issue, if the client does not know what is entailed in your service. While you can provide the site, they will need to deal with shipping themselves, unless otherwise specified.
